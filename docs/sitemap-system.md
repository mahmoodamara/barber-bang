# Production Sitemap System

## What Was Implemented

This project now uses a production-grade dynamic sitemap layer for Google crawl/index performance:

- `GET /sitemap.xml`:
  - Sitemap index.
  - Automatically lists split child sitemaps.
- `GET /sitemaps/:section-:page.xml`:
  - Sectioned child sitemaps with chunking.
  - Supported sections:
    - `static`
    - `products`
    - `categories`
    - `brands`
    - `pages`
    - `blog`
- `GET /robots.txt`:
  - Disallows admin/auth/cart/checkout and duplicate filter query patterns.
  - References canonical sitemap index URL.
- Legacy compatibility redirects:
  - `/sitemap-products.xml`
  - `/sitemap-categories.xml`
  - `/sitemap-brands.xml`
  - `/sitemap-pages.xml`
  - `/sitemap-blog.xml`
  - `/sitemap-static.xml`

## File Structure

- `server/src/services/sitemap.service.js`
  - Core sitemap architecture.
  - Data-source adapters (products/categories/brands/content/static).
  - Canonical URL construction.
  - Hreflang alternate generation (`xhtml:link`).
  - URL exclusion/noindex filtering.
  - Sitemap index + child XML builders.
  - In-memory cache and chunk metadata.
- `server/src/routes/seo.static.routes.js`
  - Express routing for `robots.txt`, sitemap index, and child sections.
- `server/.env.example`
  - Added sitemap/robots config variables.
- `frontend/netlify.toml`
  - Reverse-proxy rules for `/sitemap.xml` and `/sitemaps/*` to backend.
- `frontend/public/robots.txt`
  - Frontend-hosted robots baseline aligned with backend sitemap strategy.
- `frontend/src/App.tsx`
  - Added `/category/:categorySlug` route support.
  - Added `/blog/:slug` route alias.
- `frontend/src/pages/ShopPage.tsx`
  - Added category slug resolution (`/category/:slug` -> category filtering).

## URL Coverage Logic

Included:
- Static storefront routes (home/shop/brands/deals/bundles/ranked pages/B2B/accessibility).
- Product detail pages from active, non-deleted products with valid slug.
- Category pages from active categories with valid slug.
- Brand pages from active product brands (deduplicated; minimum product threshold configurable).
- CMS pages (`ContentPage`) split into:
  - `pages`: normal content pages.
  - `blog`: slugs matching configured blog regex/list.

Excluded:
- Admin/auth/cart/checkout/account/wishlist and other low-value/private paths.
- Filtered/faceted/search-parameter URL patterns in robots.
- Documents flagged with noindex-style fields when present.
- Duplicate URLs (deduped before XML output).

## Canonical & Multilingual Strategy

- Sitemap `loc` always uses a canonical clean URL (no faceted query URLs).
- Optional hreflang alternates are added via `xhtml:link`:
  - `he`
  - `ar`
  - `x-default`
- Language alternates use configurable query key (`SITEMAP_LANG_QUERY_PARAM`, default `lang`).

## Dynamic `lastmod`

`lastmod` is pulled from real data whenever available:
- `Product.updatedAt`
- `Category.updatedAt`
- Brand sitemap uses max product `updatedAt` per brand.
- `ContentPage.updatedAt`
- Static routes use SiteSettings/global freshness fallback.

## Scalability Design

- Split files via `SITEMAP_MAX_URLS_PER_FILE` (default `5000`, protocol max `50000`).
- Cached manifest + generated XML (`SITEMAP_CACHE_SECONDS`, default `300`).
- Child files are generated by section/page, not as one monolithic document.

## Configuration

Use these env vars in backend:

- `SITEMAP_MAX_URLS_PER_FILE=5000`
- `SITEMAP_CACHE_SECONDS=300`
- `SITEMAP_BRAND_MIN_PRODUCTS=1`
- `SITEMAP_HREFLANG_ENABLED=true`
- `SITEMAP_HREFLANG_LANGS=he,ar`
- `SITEMAP_LANG_QUERY_PARAM=lang`
- `SITEMAP_CONTENT_PAGE_PATH_PREFIX=/page`
- `SITEMAP_BLOG_PAGE_PATH_PREFIX=/page` (or `/blog` if desired)
- `SITEMAP_BLOG_SLUG_REGEX=^(blog|post|article|news)-`
- `SITEMAP_BLOG_SLUGS=`
- `SITEMAP_EXCLUDED_CONTENT_SLUGS=`
- `SITEMAP_EXCLUDED_STATIC_PATHS=`
- `SITEMAP_EXTRA_STATIC_PATHS=`
- `SITEMAP_EXCLUDE_PATH_PREFIXES=`
- `SITEMAP_ROBOTS_EXTRA_QUERY_DISALLOW=`

## Hosting Integration (React + Netlify + Express API)

If frontend is hosted on Netlify and backend on Render:
- Netlify now proxies:
  - `/sitemap.xml` -> backend `/sitemap.xml`
  - `/sitemaps/*` -> backend `/sitemaps/*`
- This keeps sitemap URLs on the primary site host while still using dynamic backend generation.

## Sample Output

### `/sitemap.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<sitemapindex xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <sitemap>
    <loc>https://barberbang.co.il/sitemaps/products-1.xml</loc>
    <lastmod>2026-03-06</lastmod>
  </sitemap>
  <sitemap>
    <loc>https://barberbang.co.il/sitemaps/categories-1.xml</loc>
    <lastmod>2026-03-06</lastmod>
  </sitemap>
</sitemapindex>
```

### `/sitemaps/products-1.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9" xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <url>
    <loc>https://barberbang.co.il/product/example-slug</loc>
    <lastmod>2026-03-06</lastmod>
    <changefreq>daily</changefreq>
    <priority>0.9</priority>
    <xhtml:link rel="alternate" hreflang="he" href="https://barberbang.co.il/product/example-slug?lang=he" />
    <xhtml:link rel="alternate" hreflang="ar" href="https://barberbang.co.il/product/example-slug?lang=ar" />
    <xhtml:link rel="alternate" hreflang="x-default" href="https://barberbang.co.il/product/example-slug" />
  </url>
</urlset>
```

## Technical SEO Notes

- Canonical routes are emitted only once per indexable entity.
- Faceted/search URLs are intentionally excluded from sitemap and discouraged via robots disallow patterns.
- Section-level splitting improves crawl budget management and update granularity.
- Keep `STORE_BASE_URL` aligned with the public primary domain (not API domain) for correct canonical sitemap `loc` values.
